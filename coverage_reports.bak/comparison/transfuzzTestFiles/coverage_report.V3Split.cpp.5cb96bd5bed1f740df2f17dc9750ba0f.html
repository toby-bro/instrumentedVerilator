16 // V3Split implements two separate transformations:
17 // splitAlwaysAll() splits large always blocks into smaller always blocks
18 // when possible (but does not change the order of statements relative
19 // to one another.)
20 //
21 // splitReorderAll() reorders statements within individual blocks
22 // to avoid delay vars when possible. It no longer splits always blocks.
23 //
24 // Both use a common base class, and common graph-building code to reflect
25 // data dependencies within an always block (the "scoreboard".)
26 //
27 // The scoreboard tracks data deps as follows:
28 //
29 // ALWAYS
30 // ASSIGN ({var} <= {cons})
31 // Record as generating var_DLY (independent of use of var), consumers
32 // ASSIGN ({var} = {cons}
33 // Record generator and consumer
34 // Any var that is only consumed can be ignored.
35 // Then we split into separate ALWAYS blocks.
36 //
37 // The scoreboard includes innards of if/else nodes also. Splitting is no
38 // longer limited to top-level statements, we can split within if-else
39 // blocks. We want to be able to split this:
40 //
41 // always @ (...) begin
42 // if (reset) begin
43 // a <= 0;
44 // b <= 0;
45 // // ... ten thousand more
46 // end
47 // else begin
48 // a <= a_in;
49 // b <= b_in;
50 // // ... ten thousand more
51 // end
52 // end
53 //
54 // ...into a separate block for each of a, b, and so on. Even though this
55 // requires duplicating the conditional many times, it's usually
56 // better. Later modules (V3Gate, V3Order) run faster if they aren't
57 // handling enormous blocks with long lists of inputs and outputs.
58 //
59 // Furthermore, the optional reorder routine can optimize this:
60 // NODEASSIGN/NODEIF/WHILE
61 // S1: ASSIGN {v1} <= 0. // Duplicate of below
62 // S2: ASSIGN {v1} <= {v0}
63 // S3: IF (...,
64 // X1: ASSIGN {v2} <= {v1}
65 // X2: ASSIGN {v3} <= {v2}
66 // We'd like to swap S2 and S3, and X1 and X2.
67 //
68 // Create a graph in split assignment order.
69 // v3 -breakable-> v3Dly --> X2 --> v2 -brk-> v2Dly -> X1 -> v1
70 // Likewise on each "upper" statement vertex
71 // v3Dly & v2Dly -> S3 -> v1 & v2
72 // v1 -brk-> v1Dly -> S2 -> v0
73 // v1Dly -> S1 -> {empty}
74 // Multiple assignments to the same variable must remain in order
75 //
76 // Also vars must not be "public" and we also scoreboard nodep->isPure()
77 //
