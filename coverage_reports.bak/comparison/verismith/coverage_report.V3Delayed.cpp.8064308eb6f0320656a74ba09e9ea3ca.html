16 // V3Delayed's Transformations:
17 //
18 // Convert AstAssignDly into temporaries an specially scheduled blocks.
19 // For the Pre/Post scheduling semantics, see V3OrderGraph.
20 //
21 // There are several "Schemes" we can choose from for implementing a
22 // non-blocking assignment (NBA), repserented by an AstAssignDly.
23 //
24 // It is assumed and required in this pass that each NBA updates at
25 // most one variable. Earlier passes should have ensured this.
26 //
27 // Each variable is associated with a single NBA scheme, that is, all
28 // NBAs targeting the same variable will use the scheme assigned to
29 // that variable. This necessitates a global analysis of all NBAs
30 // before any decision can be made on how to handle them.
31 //
32 // The algorithm proceeds in 3 steps.
33 // 1. Gather all AstAssignDly non-blocking assignments (NBAs) in the
34 // whole design. Note usage context of variables updated by these NBAs.
35 // This is implemented in the 'visit' methods
36 // 2. For each variable that is the target of an NBA, decide which of
37 // the possible conversion schemes to use, based on info gathered in
38 // step 1.
39 // This is implemented in the 'chooseScheme' method
40 // 3. For each NBA gathered in step 1, convert it based on the scheme
41 // selected in step 2.
42 // This is implemented in the 'prepare*'/'convert*' methods. The
43 // 'prepare*' methods do the parts common for all NBAs updating
44 // the given variable. The 'convert*' methods then convert each
45 // AstAssignDly separately
46 //
47 // These are the possible NBA Schemes:
48 // "Shadow variable" scheme. Used for non-unpackedarray target
49 // variables in synthesizeable code. E.g.:
50 // LHS <= RHS;
51 // is converted to:
52 // - Add new "Pre-scheduled" logic:
53 // __Vdly__LHS = LHS;
54 // - In the original logic, replace the target variable 'LHS' with '__Vdly__LHS' variables:
55 // __Vdly__LHS = RHS;
56 // - Add new "Post-scheduled" logic:
57 // LHS = __Vdly__LHS;
58 //
59 // "Shared flag" scheme. Used for unpacked array target variables
60 // in synthesizeable code. E.g.:
61 // LHS[idxa][idxb] <= RHS
62 // is converted to:
63 // - Add new "Pre-scheduled" logic:
64 // __VdlySet__LHS = 0;
65 // - In the original logic, replace the AstAssignDelay with:
66 // __VdlySet__LHS = 1;
67 // __VdlyDim0__LHS = idxa;
68 // __VdlyDim1__LHS = idxb;
69 // __VdlyVal__LHS = RHS;
70 // - Add new "Post-scheduled" logic:
71 // if (__VdlySet__LHS) a[__VdlyDim0__LHS][__VdlyDim1__LHS] = __VdlyVal__LHS;
72 // Multiple consecutive NBAs of compatible form can share the same __VdlySet* flag
73 //
74 // "Unique flag" scheme. Used for all variables updated by NBAs
75 // in suspendable processees or forks. E.g.:
76 // #1 LHS <= RHS;
77 // is converted to:
78 // - In the original logic, replace the AstAssignDelay with:
79 // __VdlySet__LHS = 1;
80 // __VdlyVal__LHS = RHS;
81 // - Add new "Post-scheduled" logic:
82 // if (__VdlySet__LHS) {
83 // __VdlySet__LHS = 0;
84 // LHS = __VdlyVal__LHS;
85 // }
86 //
87 // The "Value Queue Whole/Partial" schemes are used for cases where the
88 // target of an assignment cannot be statically determined, for example,
89 // with an array LHS in a loop:
90 // LHS[idxa][idxb] <= RHS
91 // is converted to:
92 // - In the original logic, replace the AstAssignDelay with:
